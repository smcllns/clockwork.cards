# Waitlist Signup (/buy) Implementation Plan

> **Status: COMPLETE** — deployed to Cloudflare Pages Feb 19 2026.

**Goal:** Add a `/buy` waitlist page with email capture stored in Cloudflare KV, accessible from the card's upsell footer.

**Architecture:** Standalone `buy.html` (not in the SPA) with inline JS for form handling. A Cloudflare Pages Function at `functions/api/waitlist.ts` handles POST, stores to KV. `buy.html` is copied to `dist/` by `build.ts` so it deploys alongside the card.

**Tech Stack:** Cloudflare Pages Functions, Cloudflare KV, plain HTML/CSS/JS (no React/Bun on this page)

### Task 2: Create the Cloudflare Pages Function

**Files:**
- Create: `functions/api/waitlist.ts`

**Step 1: Create the functions directory and file**

```bash
mkdir -p functions/api
```

**Step 2: Write the function**

`functions/api/waitlist.ts`:
```typescript
interface Env {
  WAITLIST: KVNamespace;
}

export const onRequestPost: PagesFunction<Env> = async ({ request, env }) => {
  let body: { email?: string };
  try {
    body = await request.json();
  } catch {
    return Response.json({ error: "Invalid JSON" }, { status: 400 });
  }

  const email = (body.email ?? "").trim().toLowerCase();
  if (!email || !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    return Response.json({ error: "Invalid email" }, { status: 400 });
  }

  const existing = await env.WAITLIST.get(`email:${email}`);
  if (existing) {
    // Silently succeed — no need to tell them they're already registered
    return Response.json({ success: true });
  }

  await env.WAITLIST.put(`email:${email}`, JSON.stringify({ signedUpAt: new Date().toISOString() }));
  return Response.json({ success: true });
};
```

Note: 409-on-duplicate was dropped in favor of silent success (Sam's call during brainstorm) — simpler UX.
